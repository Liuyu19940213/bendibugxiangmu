    PRD：Pixelle-Video 桌面应用¶

    项目代号：pixelle_video_desktop
    技术栈：Tauri 2.0 + Vue3 + Shadcn-Vue + Tailwind CSS · 后端 Python
    版本：v1.0
    语言：中文

    1. 产品目标¶

    将 Pixelle-Video 引擎封装为桌面批量创作工具，让书单号运营者「调好参数 → 一键启动 → 睡一觉 → 醒来发布」——以最小操作成本将一批书籍文案批量转化为沉稳风格的 8–13 分钟短视频。

    2. 用户故事¶

    # 场景 用户故事
    1 批量出片 作为书单号运营者，我手上有 50 本书的待推广清单，我希望一次性导入书名和文案，统一设置配音、BGM、画面风格后点击「批量运行」，第二天醒来所有视频已生成完毕，失败的书自动跳过并标注原因。
    2 精调单篇 作为内容创作者，某本重点推广书籍需要单独打磨——我要进入精细模式，对洗稿结果、配音节奏、配图选择逐环节预览和调整，满意后再生成最终视频。
    3 素材积累复用 作为长期运营者，我希望每次用过的图片、BGM、文案自动入库归类，下次做同类书籍时自动推荐匹配素材，同一素材 30 天内不重复出现，避免观众审美疲劳。

    3. 需求池¶

    P0 — 必须实现¶

    需求 功能描述 验收标准 页面
    批量创作页 应用默认首页。顶部全局配置栏（配音/BGM/时长/风格），下方 N 个模块卡片（每卡 = 书名 + 文案 + 洗稿参数）。支持 TXT 批量导入、手动增删、启用/禁用开关、同名书配置联动、按书名自动排序。 1. TXT 导入：空行切段 → 首行 《书名》 匹配 → 其余为文案；未匹配段落单独提示不丢弃。2. 全局参数修改后所有未覆盖模块继承新值。3. 空模块运行时自动跳过不报错。4. 同名书修改任一卡片配置，所有同名卡片联动。 批量创作
    批量运行引擎 串行排队执行所有已启用模块。单模块失败跳过继续、进度持久化。 1. 运行中展示总进度条 + 当前处理书名 + 已完成/失败计数。2. 完成后弹出汇总：成功 N、失败 M（含原因）、跳过 K。3. 未完成任务下次启动弹窗：「检测到上次未完成任务，是否继续？」用户可选清空/继续。 批量创作
    洗稿集成 复用「再睡一夏」方案：整篇文案一次喂 LLM、单次流式输出、锁死结构骨架（人设/钩子/情绪节奏/书名卖点）只换表达血肉。与素材库联动匹配同名素材（0–10 篇可调）。SHA256 + SequenceMatcher ≥ 92% 去重。 1. 输出 3000–4500 纯中文字符（[\u4e00-\u9fff]）。2. Prompt 注入字数目标。3. 流式输出实时展示。4. 去重逻辑生效。 引擎层
    精细模式 单本书逐环节创作流程：洗稿 → 配音 → 配图 → 生成视频。每步可单独修改参数后重跑、预览中间结果。 1. 步骤条高亮当前环节。2. 洗稿后查看文案、配音后试听、配图后预览图片、视频后播放预览。3. 每步参数面板可调。4. 最终视频输出到指定目录。 精细模式
    素材库-图片 分两大图库独立管理：①预设图库（用户上传 jpg/png/webp，5 类预设分类：自然风景 / 演播室 / 书房 / 抽象意境 / 未分类）+ ②AI 图库（AI 生成图片自动归入，单独标签管理）。两库互不混合。AI 自动标签 + 手动标签。30 天冷却期规则。Ken Burns 效果开关。 1. 拖拽导入即时入库到预设图库。2. 分类筛选各自独立生效。3. AI 生成图片自动归入 AI 图库。4. 同一图片 30 天内不重复推荐给同一模块。5. Ken Burns 开关控制缓慢缩放平移。 素材库
    素材库-BGM 激昂/宁静两类，每类内置 3–5 首免费曲目。用户可上传 mp3/wav。淡入淡出 2s/3s。 1. 试听预览正常。2. 上传格式校验。3. 淡入淡出参数可调。 素材库
    素材库-文案 题材分类：书籍 / 播客 / 口播 / 访谈 + 自定义。每条存原文 + 情绪节奏 JSON。 1. 按题材筛选。2. 做视频时同题材自动匹配已有情绪节奏。 素材库
    设置页 LLM API 配置（地址/Key/模型，兼容 OpenAI 接口，支持通义千问/DeepSeek/豆包）。配音 API（MiMo）配置 + 本地 IndexTTS-2 via ComfyUI 切换。ComfyUI 一键下载按钮。输出路径选择。 1. API 连通性测试按钮。2. 本地/API 切换即时生效。3. ComfyUI 下载有进度提示。4. 修改后无需重启。 设置
    应用布局 左侧 160px 深色导航栏（shadcn-vue-admin 风格）。路由：批量创作（默认首页）→ 精细模式 → 模板画板 → 素材库 → 文案库 → 设置。设置固定在导航底部、与其他项用分割线隔开。 1. 导航点击切换路由。2. 默认首页为批量创作。3. 设置项始终在底部。 全局

    P1 — 应该实现¶

    需求 功能描述 验收标准 页面
    模板调参画板 左侧实时视频预览 + 右侧参数控件面板。调整参数后预览刷新（2 秒防抖）。所见即所得。 1. 视频播放器展示 15–30 秒预览片段。2. 参数面板：文字位置/大小/颜色、字幕样式、转场效果、Ken Burns 幅度。3. 「另存为自定义模板」不覆盖原模板。 模板画板
    情绪节奏系统 爆款视频上传 → Whisper 转文字 + 时间戳 → 按句切段 → 标点推断情绪 → 入库。做视频时文案标点自动推断情绪 → 手动微调 → 存为模板。文案库同题材自动匹配已有情绪节奏。 1. 视频上传后自动转录分段。2. 情绪曲线可视化。3. 手动微调后保存。4. 同题材自动推荐。 精细模式 / 素材库
    文案库独立页 全文搜索、按题材/标签筛选、批量导入导出（JSON/CSV）、情绪节奏波形图预览。 1. 搜索 < 1s 响应。2. 导出格式正确。3. 波形图渲染正常。 文案库
    视频号发布对接 视频生成后一键发布到微信视频号平台。对接视频号开放平台 API，支持标题/描述/标签填写、定时发布、发布状态查询。 1. 视频号授权登录正常。2. 发布参数（标题/描述/标签）可配置。3. 发布后返回状态可查询。4. 发布失败有明确错误提示。 批量创作 / 精细模式

    P2 — 可以后续¶

    需求 功能描述 验收标准 页面
    模块模板 保存一组模块配置为模板，一键套用。 模板保存/加载/删除正常。 批量创作
    定时任务 设定时间自动开始批量运行。 到达设定时间后自动启动。 批量创作
    运行历史 查看每次批量运行的详细日志。 历史列表 + 日志详情。 批量创作
    多语言 界面中/英切换。 切换后所有文案即时更新。 全局

    4. UI 设计要点¶

    4.1 全局布局¶

    ┌──────┬────────────────────────────────────────────────┐
│ 导航  │                                                │
│160px │              路由内容区                          │
│ 📚批量│                                                │
│ 🎯精细│                                                │
│ 🎨画板│                                                │
│ 📦素材│                                                │
│ 📝文案│                                                │
│──────│                                                │
│ ⚙设置│                                                │
└──────┴────────────────────────────────────────────────┘
    • 左侧导航：固定 160px，深色背景，图标 + 文字菜单。设置项固定在底部，与上方用分割线隔开。
    • 设计原则：大字体、高对比度（面向中老年用户），暗色主题优先（参考 shadcn-vue-admin），操作反馈清晰（loading / 成功 / 失败），防误操作（删除二次确认、运行中禁用配置修改）。

    5. 已确认问题¶

    # 问题 结论
    1 视频号发布对接 ✅ 纳入需求（见 P1）。需要对接视频号开放平台 API 实现一键发布。
    2 「再睡一夏」Prompt 迁移 ✅ 直接复用 D:\wudiyuanchuang\ 成熟 Prompt 逻辑，整篇一次性喂 LLM、流式输出。
    3 ComfyUI 本地部署门槛 ✅ 用户 GPU RTX 4060Ti 8G，满足 ≥6GB 要求，配音本地方案纳入首版。
    4 视频分辨率与编码 ✅ 1080p / 30fps / H.264 编码，标准抖音输出分辨率。
    5 配图版权 ✅ 用户决定无视，不做免责声明。
    6 图片库分类 ✅ AI 生成图库和预设图库分开管理，不混在同一分类。
